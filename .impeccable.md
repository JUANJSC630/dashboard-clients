## Design Context

### Users
Agency owners and independent freelancers managing hosting clients, sites, billing, and uptime monitoring. Used primarily during business hours in an office context. The tool is operational and task-focused — users open it to check status, manage clients, and track billing.

### Brand Personality
**Three words**: precise, operational, calm

Word Code is a professional utility tool — like a well-made industrial instrument. No decoration for decoration's sake. Every element earns its place.

### Aesthetic Direction
- **Theme**: Supports both light and dark (system preference)
- **Font**: Onest — clean geometric humanist sans, precise yet approachable. Not in the AI monoculture list.
- **Color**: Existing shadcn/ui token system preserved. Neutrals lead; destructive red only for real alerts (incidents, overdue). No accent gradients.
- **Density**: Medium — enough white space to breathe, not so sparse it wastes screen real estate on a dashboard
- **Anti-references**: No glassmorphism, no left-border stripes on items, no gradient text, no hero-metric templates with glowing accents

### Design Principles
1. **Data first** — numbers are the hero, not decorative icons or background gradients
2. **Status is binary** — if nothing is wrong, show nothing alarming. Alert colors only when actionable
3. **Navigation is ambient** — sidebar is present but not competing for attention; active states are subtle
4. **Forms are calm** — auth split-screen: dark brand context on left, clean form on right
5. **No redundancy** — remove duplicate labels, repeated headings, "View all" links styled as web-blue hyperlinks
